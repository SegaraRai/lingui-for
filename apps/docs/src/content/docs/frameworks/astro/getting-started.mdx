## Install the packages

<Tabs syncKey="package-manager">
	<TabItem label="pnpm">

```sh
pnpm add @lingui/core lingui-for-astro
pnpm add -D @lingui/cli @lingui/conf unplugin-lingui-macro
```

    </TabItem>
    <TabItem label="npm">

```sh
npm install @lingui/core lingui-for-astro
npm install -D @lingui/cli @lingui/conf unplugin-lingui-macro
```

    </TabItem>
    <TabItem label="yarn">

```sh
yarn add @lingui/core lingui-for-astro
yarn add -D @lingui/cli @lingui/conf unplugin-lingui-macro
```

    </TabItem>

</Tabs>

<Aside>

`unplugin-lingui-macro` is only needed if you use Lingui macros in plain JavaScript or TypeScript
files. If you only use macros in `.astro` files, you can skip installing this package and remove it
from `vite.plugins` in `astro.config.ts`.

</Aside>

## Configure Astro

```ts title="astro.config.ts"
import { defineConfig } from "astro/config";
import linguiForAstro from "lingui-for-astro/integration";
import linguiMacro from "unplugin-lingui-macro/vite";

export default defineConfig({
  integrations: [linguiForAstro()],
  vite: {
    plugins: [linguiMacro()],
  },
});
```

<Aside>

Astro defaults to static output. The middleware example below is the request-aware setup for
`server` or `hybrid` output. If your site is fully static, see
[Request Context](/frameworks/astro/request-context) for the frontmatter-based setup instead.

</Aside>

## Configure Lingui

```ts title="lingui.config.ts"
import { defineConfig } from "@lingui/conf";
import { astroExtractor } from "lingui-for-astro/extractor";

export default defineConfig({
  locales: ["en", "ja"],
  sourceLocale: "en",
  catalogs: [{ path: "src/lib/i18n/locales/{locale}" }],
  extractors: [astroExtractor],
});
```

## Initialize Lingui for your output mode

For `server` and `hybrid` output, call `setLinguiContext` from middleware before any page renders.
This is where you load the compiled catalog for the current request.

```ts title="src/middleware.ts"
import { defineMiddleware } from "astro:middleware";
import { setupI18n } from "@lingui/core";
import { setLinguiContext } from "lingui-for-astro";
import { catalog } from "./lib/i18n/catalog";

export const onRequest = defineMiddleware((context, next) => {
  const locale = resolveLocale(context); // your locale resolution logic
  const i18n = setupI18n({ locale, messages: catalog });
  setLinguiContext(context.locals, i18n);
  return next();
});
```

<Aside>

`catalog` is a locale-keyed object exported from a shared setup file. See
[Load Compiled Catalogs](/guides/load-compiled-catalogs) for how to create that file and choose
between bundling all locales upfront or loading them on demand.

</Aside>

## Use macros in `.astro`

```astro title="src/pages/index.astro"
---
import { Trans, t } from "lingui-for-astro/macro";
---

<h1>{t`Hello from Astro`}</h1>
<p><Trans>Macro-first translation in Astro</Trans></p>
```
