`msg` and `defineMessage` define message descriptors without translating them immediately.
`msg` is an alias for `defineMessage`; they are functionally identical.

They are the right tools when the message needs to be stored, exported, reused across files, or
translated later by a different renderer or runtime boundary.

For the framework-agnostic Lingui semantics, see the
[official `msg` and `defineMessage` reference](https://lingui.dev/ref/macro#definemessage).

## Usage

<Tabs syncKey="framework">
  <TabItem icon="seti:svelte" label="Svelte">

```svelte
<script lang="ts">
  import { defineMessage, msg, t } from "lingui-for-svelte/macro";

  const welcome = msg`Welcome back`;

  const submitLabel = defineMessage({
    id: "form.submit",
    message: "Submit",
  });
</script>

<p>{$t(welcome)}</p>
```

const welcome = msg`Welcome back`;

  message: "Submit",
});

const label = t(welcome);

<Aside type="tip">

Descriptors can also be defined and exported from a separate file. To use them in plain JavaScript
or TypeScript:

- Import macros from `@lingui/core/macro`.
- Transform with `unplugin-lingui-macro`.

See [Plain JS/TS Setup](/guides/plain-js-ts) for the full setup and the recommended authoring
pattern.

</Aside>

## When they are a good fit

- the translation should happen later, not at definition time
- the same descriptor will be reused across files or renderers
- you want plain TypeScript or JavaScript to stay close to official Lingui Core patterns

## Recommended rule

For plain `.js` and `.ts` files, prefer the official `@lingui/core/macro` package. Framework-specific
packages are for framework syntax, not for generic descriptor authoring.
